# ComfyUI PromptPalette

## プロジェクト概要

ComfyUI-PromptPaletteは、マウス操作で素早いプロンプト編集を可能にするComfyUI向けのカスタムノードです。
チェックボックスによるフレーズの有効/無効切り替えや、ウェイト調整ボタンといった、インタラクティブなプロンプト編集機能を提供します。

## 開発環境

- ComfyUI
  - Nodes 1.0およびNodes 2.0対応
- Python
- Node.js

## コーディング規約

- コードコメントは英語
- ログ出力は英語

## コミット規約

- コミットメッセージは英語

## インストール方法

標準的な ComfyUI カスタムノードの導入手順に従い、`custom_nodes` ディレクトリへ配置する。
追加のセットアップや依存関係は不要。

## プロジェクト構成

ComfyUIのカスタムノード構成に従う。

- `__init__.py`: ノードのマッピングと Web ディレクトリをインポート/エクスポートするエントリポイント
- `nodes.py`: テキスト入力を処理するバックエンド（Python）
- `web/index.js`: フロントエンドのエントリポイント（JavaScript）
- `web/canvas_ui.js`: Nodes 1.0用のフロントエンド（Canvas描画）
- `web/dom_ui.js`: Nodes 2.0用のフロントエンド（DOM描画）
- `web/ui_utils.js`: UI関連の共通ロジック
- `web/line.js`: テキスト処理の共通ロジック
- `pyproject.toml`: Comfy Registry 公開のためのメタデータ

## 機能

### ノードの入出力

以下の通り整形した文字列を出力する：

- ノード内に書かれた複数行テキストから、空行と `//` で始まるコメント行を除外
- 行内の `//` 以降はインラインコメントとして除去
- 除去されず残った行の末尾にカンマ (,) を追加。ただし既に末尾カンマがあれば追加しない。
- `prefix` 入力があれば先頭に追加。改行ありで結合する。

### 編集モードと表示モードの切り替え

- ノード下部の `Edit` / `Save` ボタンで編集モードと表示モードをトグル
- デフォルトは表示モード

### 編集モード時のUI

- 標準の複数行テキストウィジェットと、ノード下部に `Save` ボタンを表示

### 表示モード時のUI

#### 用語定義
- 「表示用テキスト」：各行のテキストから、コメントプレフィックスおよびウェイトの数字、ウェイトの括弧、ウェイトのコロン、末尾カンマを除去したテキストを表示用テキストとする。インラインコメントは表示用テキストに含まれる。
- 「フレーズテキスト」：各行のテキストから、コメントプレフィックスおよびインラインコメント、ウェイトの数字、ウェイトの括弧、ウェイトのコロン、末尾カンマを除去したテキストをフレーズテキストと呼ぶ。

#### Nodes 1.0と2.0共通
- 行ごとに「チェックボックス」「表示用テキスト」「ウェイトの数字」「-ボタン」「+ボタン」を描画
  - チェックボックスの有効/無効に応じて表示用テキストを色分け
  - ウェイトが 1.0 以外の場合、その行の表示用テキストを太字表示
  - ウェイトが 1.0 の場合、ウェイトの数字を非表示
- フレーズテキストが空文字や空白文字になる場合、その行は空行として表示する。例えばコメントプレフィックスのみの行や、コメントプレフィックスとインラインコメントのみの行などが該当する。
- ノード下部に `Edit` ボタンを表示
- テキストが空の場合はノードの中央に "No Text" と表示
- 行数に応じてノード高を自動調整
- ノードが折りたたまれている場合は描画をスキップ

#### Nodes 1.0の場合
- Canvasで UI を描画
- クリック処理は座標ベースの `clickableAreas` により判定

#### Nodes 2.0の場合
- DOMでUIを描画

### チェックボックスによる行頭コメントの切り替え

- チェックボックスがオフの場合、その行に行頭コメント `//` を付ける
- チェックボックスがオンの場合、その行の行頭コメントを削除する

### ウェイト調整

- `(phrase:1.5)` のようなウェイト表記を解析し、1.0 以外のウェイトを表示
- ± ボタンで 0.1 刻みの増減
- ウェイトの範囲は 0.1〜2.0
- 末尾カンマを除去してからウェイト表記を再構成
- 1.0 に戻すとウェイト表記自体を外して元のテキストに戻す
- 上限超過の値は増加を拒否し、減少時のみ上限にクランプ
- 行頭コメント行やインラインコメント付き行でも増減可能

### テーマ連携

- ComfyUI の CSS 変数から色を取得し、描画に利用
- ライト/ダーク両テーマに対応
- パフォーマンスのためのカラーキャッシュ
- 3桁の16進カラーの展開に対応

### 拡張登録

- ComfyUI 拡張として登録し、PromptPalette ノードの生成/描画にフック
- `beforeRegisterNodeDef` で PromptPalette ノードの挙動を差し替え

## ComfyUIの実行方法

ComfyUI portableのcustom_nodesフォルダ内にComfyUI-PromptPaletteフォルダを配置している。
ワーキングディレクトリがComfyUI-PromptPaletteフォルダとすると、以下のコマンドでComfyUIを起動する。

```
..\..\..\python_embeded\python.exe -u -s ..\..\..\ComfyUI\main.py --windows-standalone-build --disable-auto-launch
```

## テスト方法

1. ComfyUIを再起動
2. MCP PlaywrightでComfyUIを開く
3. 画面上部の+ボタンで新しいWorkflowを作成
4. 画面中央付近をダブルクリックしてノードを検索
5. "PromptPalette" と入力してEnter
6. 画面中央にPromptPaletteノードが追加されたことを確認
7. PromptPaletteノードのEditボタンを押す
8. PromptPaletteノードのテキストエリアに以下の行を追加
```
aaa
(bbb:0.9) // BBB
// (abc:1.2)
//
```
9. PromptPaletteノードのSaveボタンを押す
10. hogeの右横の+ボタンを2回押す
11. hogeの右横の数値が1.2になったことを確認
